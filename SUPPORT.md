# Support

Use the right channel so issues are actionable and easy to triage.

## Before opening an issue

Read these first:

- [README.md](README.md)
- [INSTALL.md](INSTALL.md)
- [KEYBINDINGS.md](KEYBINDINGS.md)
- [PERFORMANCE.md](PERFORMANCE.md)
- [SECURITY.md](SECURITY.md)

If the question is about how the config is structured internally, [BRAIN.md](BRAIN.md)
contains the design notes and module map.

## Which template to use

- **Bug report**: something is broken or regressed.
- **Feature request**: you want new behavior or a better workflow.
- **Usage help**: you are trying to understand how to use an existing feature.

## Release downloads

The supported binary-style distribution for this project is the release asset
zip published on the [GitHub Releases](https://github.com/ashut0shk/nvim-config/releases) page.
Each tagged release publishes:

- `nvim-config-<tag>.zip`
- `nvim-config-<tag>.zip.sha256`

This repository is a Neovim config, not a library or CLI package, so GitHub
Packages is not used as a distribution channel.

Neovim 0.12 — Keybindings Reference

Leader: \ · Vim grammar: untouched · No built-in collisions

Architecture

Custom maps only use characters that are already prefix keys in Vim: g [ ] \ (leader) <C-*> <S-*> <A-*>

No built-in single-key command is ever overridden with a multi-key combo.

Namespaces (always have extensions — never standalone): \f* Find · \g* Grep · \s* Search/QF · \b* Buffer · \c* Code · \h* Hunk · \t* Toggle · \l* LSP · \G* Git

Instant (single key after \ — no extensions exist, 0ms): \w \q \Q \e \T \j \J \1 \v \H \r \u \D \<leader>

g-prefix (Vim native prefix — zero added latency): go find files · ga live grep · gz revert buffer

Fast Access — Daily Operations

Key Action
go Find files (fzf, precached, respects ignore)
ga Live grep (fzf + rg, respects ignore)
\e Toggle sidebar with current file revealed
gz Revert file to original (buffer → disk)
<A-d> Diff file vs HEAD (side-by-side, ignore whitespace)
